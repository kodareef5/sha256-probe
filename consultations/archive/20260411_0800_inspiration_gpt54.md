- if frontier size stays small, the algorithm works.

# Carry-conditioned elimination at N=8

Build the explicit-carry model and eliminate data vars as far as possible.

Minimum viable version:
1. expose all carry vars,
2. separate a-path carries,
3. for each a-path assignment, solve the affine shell,
4. record residual carry count and syndrome.

## What to measure
- survivor count after a-path solve,
- rank distribution of the affine system,
- number of residual T1 carry vars,
- distinct syndrome values per surviving a-path class.

## Why this matters
This is the direct bridge from theorem to algorithm.

## Hardware
- 24-core server: parallel a-path enumeration
- 10-core Mac: symbolic preprocessing and validation
- GPU: not needed

## Success criterion
If a-path enumeration collapses to a few hundred or few thousand viable classes, your solver architecture is validated.

# 8. Specific variant approaches to try

A few very concrete ones beyond the top 3:

## A. Myhill–Nerode state-count experiment
For N=8, define round-prefix carry states. Two prefixes are equivalent if they admit the same set of completions to a collision or same set of possible residual syndromes. Merge by hashed completion signature.

If the number of equivalence classes is tiny, that is your automaton.

## B. MUS/MCS over schedule bits
Instead of only testing all pairs:
- enumerate minimal correction sets for the schedule syndrome,
- compare against observed syndrome-support orbit.

## C. Triple-bit relaxation at N=8/N=10
Once you know the syndrome set \(E\), test whether SAT subsets equal support-hit subsets.

## D. Output-redundancy rank test at N=8
After carry-conditioning, compute rank of the 8 output equality equations. If `h` is truly redundant structurally, rank should drop to 7 generically on the solution-supporting carry classes.

## E. Low-bit exact cone in full 32-bit SHA-256
Build exact 32-bit dependency cones for:
- `d[0..k]`,
- `h[0..k]`,
- schedule residual bits around actual \(\varepsilon\)-support.

That gives real SHA-256 statements instead of mini-SHA surrogates.

# 9. What I would stop doing now

1. **Stop trying to explain critical pairs by raw sigma1 geometry alone.**  
   Compute the syndrome orbit first.

2. **Stop using output HW as a primary score.**  
   Replace with syndrome weight + carry consistency score.

3. **Stop spending serious cycles on single-bit annihilator/correlation hunting.**  
   Those are not on the main path anymore.

4. **Stop treating mini-SHA scaling as evidence for N=32 behavior.**  
   Use it only to prototype algorithms.

5. **Stop calling the entropy observation a standalone theorem without the injectivity wording.**  
   The mathematically clean statement is:
   > the projection from the collision set to carry signatures is injective at N=4,6,8.

That is sharper and more defensible.
