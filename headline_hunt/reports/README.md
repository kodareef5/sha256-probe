# Reports

- `dashboard.md` — generated by `infra/summarize_runs.py`. Per-bet rollup of
  runs, CPU spend, audit failures. Run weekly; commit the result.
- `decision_memos/` — one memo per non-trivial direction change (e.g., "we are
  now prioritizing block2_wang over true_sr61_n32 because X"). Date-prefixed.
- `weekly/` — week-of summaries. Optional but useful for multi-machine coordination.

Format for a decision memo:

```
# <YYYY-MM-DD> — <short title>

## Decision

## Why now

## What changes (bets, priorities, ownership)

## What does not change

## Author / machine
```
