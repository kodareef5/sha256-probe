# Per-paper notes

When you read a paper from `../../registry/literature.yaml`, write your notes
here as `<paper-id>.md`. Update the paper's `read_status` to `read` and fill in
its `key_takeaway` field.

Note format suggestion:

```
# <paper id> — <citation>

## TL;DR (3 lines)

## Most relevant findings to this project

## Action items for our bets

## Reservations / caveats
```

Notes go in git so other machines can read them on pull.
