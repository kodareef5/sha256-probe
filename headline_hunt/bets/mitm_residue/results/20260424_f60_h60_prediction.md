# f60 and h60 prediction VERIFIED — g60 needs marginal-distribution model

Extension of `20260424_algebraic_prediction.md`. Two of the three round-60 hard-residue
register positions have closed-form O(1) prediction; the third (g60) does not.

## Closed-form predictions (verified empirical 4/4 each)

**dh60** (h-register at round 60 = de57):
- `Δe57 = dd56_mod + dh56 + dSigma1(e56) + dCh(e56,f56,g56) − cw57` (mod 2^32)
- This simplifies via `cw57 = dh56 + dSigma1 + dCh + dT2_56` to:
- `Δe57 = dd56 − dT2_56`
- All terms candidate constants (round-56 state). O(1) compute.

**df60** (f-register at round 60 = de59):
- de59 = dd58 + dT1_59. Cascade gives dT1_59 = −dT2_at_round_59 = 0 (because round-58 state has da=db=dc=0 zero-diff under cascade).
- So `Δe59 = dd58 = dc57 = db56_mod` (modular)
- Pure candidate constant: `(state1[1] − state2[1]) mod 2^32`.

For both, predictor:
```
uniform_bits(Δ) = {i ∈ [1,31] : (Δ mod 2^i) / 2^i ∈ [0.45, 0.55]}
```

## Why g60 doesn't have a clean closed form

g60 = de58 = dd57 + dT1_58. dT1_58 = −dT2_at_round_58 (cascade). dT2_at_round_58 uses round-57 state:
- da57 = 0 (cascade ✓)
- db57 = da56 = 0 (cascade ✓)
- dc57 = db56 ≠ 0 (candidate constant; cascade has not propagated this far)

So `dT2_58 = dMaj(a57, b57, c57)` where (a57, b57, c57) values depend on w1_57. Different w1_57 produces different a57 → different dMaj → different cw58 → different Δe58.

Empirically: 5 random (w1_57, w1_58) samples produce 5 distinct Δe58 values. So Δe58 is NOT a candidate constant.

The empirical g60 hard-bit positions ARE stable (per the cross-candidate sweep), which means the MARGINAL distribution of Δe58 over uniform random w1_57 has a stationary structure that yields fixed bit-uniform positions. Deriving that marginal analytically requires more work — likely involves the structure of MAJ over differential inputs.

## Open: g60 marginal-distribution prediction

The structure to exploit:
- Δe58 = dd57 + (−dT2_58)
- dd57 = db56_mod (candidate constant, contributes its own predictable bits)
- −dT2_58 = −dMaj(a57, b57, c57) where (a57, b57, c57) varies with w1_57

For a CDF analysis: need the distribution of dMaj(a57, b57, c57) over uniform random w1_57. Since b57 = a56 (constant), c57 = b56 (constant), and a57 = T2_56 + T1_56 = const + w1_57 (modular shifted), a57 is uniform mod 2^32. So dMaj has a fixed candidate-determined distribution we'd need to characterize.

Then Δe58 = dd57 − dMaj(a57, b57, c57) where dd57 is fixed and the second term has a known distribution. The R XOR (R + Δe58) where Δe58 is itself a random variable produces the observed g60 bit-frequency pattern.

Likely a 1-day exercise to derive the formula; saving for later.

## Reproduce

```python
# h60 prediction
de57_mod = (dd56 + dh + dSig1 + dCh - cw57) & MASK
# alternatively: de57_mod = (dd56 - dT2_56) & MASK
h60_uniform = [i for i in range(1,32)
               if 0.45 <= ((de57_mod & ((1<<i)-1)) / (1<<i)) <= 0.55]

# f60 prediction
de59_mod = db56_mod  # = (s1[1] - s2[1]) & MASK
f60_uniform = [i for i in range(1,32)
               if 0.45 <= ((de59_mod & ((1<<i)-1)) / (1<<i)) <= 0.55]
```
