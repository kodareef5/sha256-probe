# chunk_mode_dp — q5 prior art audit

The bet is "chunk-mode DP with mode variables on future-completion state." The hypothesis: a DP state based on future-completion (boundary modes + canonical residual context) is compact enough to break the carry-state-injectivity barrier that closed raw carry DP.

## What chunk_mode_dp specifically needs (not in q5)

GPT-5.5's framing of "future-completion mode variables" is NOT prototyped:

- **Mode variable**: a discrete state indicating which of finitely many "completion modes" the partial state is in. The bet's hope is that the number of distinct modes is polynomial in N, even though the carry state alone is near-injective (89-99% of 2^{4N}).
- **Future-completion quotient**: the equivalence class of partial states that can extend to the same set of full collisions. The bet's claim is that this quotient is much smaller than the raw carry state.

The closest analog is `bitserial_quotient_dp.c` which is the closed negative result. ae3.c uses a different state (affine forms over GF(2)).

## Conclusion

`chunk_mode_dp` Phase 1 (designing the mode-variable set) cannot be skipped by reusing q5 code. The bet genuinely requires fresh design work: define what "mode" means concretely, characterize the modes for cascade collisions at small N, prototype the DP at N=8 to recover the known 260 collisions with mode-augmented compact state.

Estimated effort: ~3-5 days of design + N=8 prototype + verification. Multi-day, not a side project.
