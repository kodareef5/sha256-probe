## Structural Interpretation

The cascade's e-register diff is nearly deterministic: 3 of 4
de values (de57, de59, de60) are CONSTANT across all w57 values.
Only de58 varies, and its image is tiny relative to the state space.

This confirms: the cascade state space is HIGHLY structured,
not random. Each round collapses the effective dimensionality.

## NOT Directly Usable for Pruning

The de58 set is the FULL IMAGE of g(w57) → de58. Since every
w57 maps to one of these 1024 values, we can't skip w57 values
based on de58 alone. Pruning requires knowing which de58 values
correspond to collision-producing w57 values.

However, if combined with carry automaton constraints (which
DEPEND on de58 value), this could enable de58-conditioned
carry search: for each of the 1024 de58 classes, run a
carry-state DP. Width of each class may be much smaller.
