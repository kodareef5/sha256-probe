## 2. Why this breaks the rotation frontier

The rotation frontier theorem killed any algorithm of the form:

- assign low bits,
- use current values only,
- prune immediately.

That fails because \(\Sigma_1(e)[k]\) depends on future bits.

But if future bits are kept as **symbolic variables**, then \(\Sigma_1\) is not a problem at all:
it is just an XOR of symbols.

The true obstruction is not rotations. It is that the nonlinear pieces:
- addition carry,
- Ch,
- Maj

must also be handled symbolically.

The right trick is to branch only on the **nonlinear control variables** that make those operators linear.

### Exact local linearizations

For one bit:

### Ch
\[
\text{Ch}(e,f,g)=g \oplus e(f\oplus g)
\]
If you branch on \(q=e\), then:
- \(q=0 \Rightarrow \text{Ch}=g\)
- \(q=1 \Rightarrow \text{Ch}=f\)

### Maj
\[
\text{Maj}(a,b,c)=c \oplus (b\oplus c)(a\oplus c)
\]
If you branch on \(p=b\oplus c\), then:
- \(p=0 \Rightarrow b=c,\ \text{Maj}=c\)
- \(p=1 \Rightarrow b\neq c,\ \text{Maj}=a\)

### Binary addition
For one binary adder bit \(x+y+\text{cin}\to z,\text{cout}\), branch on the **3-valued mode**
\[
m \in \{00,\ 01,\ 11\}
\]
meaning:
- \(m=00\): \(x=0,\ y=0\)
- \(m=01\): \(x\oplus y=1\)
- \(m=11\): \(x=1,\ y=1\)

Then:
- \(z = \text{parity}(m)\oplus \text{cin}\)
- \(\text{cout} = 0,\text{cin},1\) respectively

and the constraints on \(x,y\) are **linear**:
- \(x=0,y=0\)
- \(x\oplus y=1\)
- \(x=1,y=1\)

So fixed local mode + fixed Ch/Maj selector bits + fixed incoming carries makes the whole slice affine over GF(2).

That is the breakthrough object.

## 6. Recurrence: count suffix completions

The cleanest implementation is memoized suffix counting.

```c
uint64_t CountSuffix(int chunk_idx, State s);
```

Interpretation:
- number of full collisions completing state `s` from chunk `chunk_idx` onward

- at end of last chunk, accept iff final registers match and all pending constraints vanish

- enumerate all feasible local symbolic transitions through the current chunk
- canonicalize each successor state
- recurse

Because the same canonical state will be reached by many prefixes, memoization collapses the search.
