# 1. Concrete algorithm to break below \(2^{4N}\)

The right object is **not** the message space. It is the set of **residual constraint signatures** after processing a prefix of bit positions.

## Core idea

After you expose carries, each bit-slice of the 7-round tail becomes:

- **nonlinear only in a small local carry interface**, and
- **linear over GF(2)** in the message bits and rotated/shifted frontier bits.

So define a DP state as:

1. **boundary carry vector** across the cut, and  
2. **RREF of the residual affine system** on the still-unassigned frontier bits.

Then scan bit positions, merge equivalent residual states, and never enumerate all \(2^{4N}\) words.

---

## State definition

Let the free bits be
\[
x_{r,k} = W_1[r][k],\quad r\in\{57,58,59,60\},\ k\in\{0,\dots,N-1\}.
\]

Using your cascade:
- \(W_2[57]\) is affine in \(W_1[57]\),
- \(W_2[58]\) is determined by \(W_1[57]\) and lower carry state,
- \(W_2[59]\) by \((W_1[57],W_1[58])\),
- \(W_2[60]\) by prior state.

So only the \(x_{r,k}\) remain as decision bits.

For a cut after processing bit positions \(0,\dots,k-1\), define:

### Boundary carry signature \(\gamma_k\)
All carry-ins to bit \(k\) for the live adders in rounds 57..63, plus the carry state for the four maps
\[
W_2[r] = W_1[r] + C_r(\cdot).
\]

You do **not** need every internal wire; only carries crossing the cut.

### Frontier variable set \(U_k\)
Bits of \(W_1[57..60]\) not yet assigned but already referenced by rotations/shifts from processed slices.

Because the maximum relevant offset is bounded (17,19,10 in full SHA-256; scaled equivalents in toy widths), \(|U_k|\) is **\(O(1)\) for fixed SHA-256**, roughly a few dozen to ~100 bits, not \(O(N)\).

### Residual affine system \(A_k u = b_k\)
Conditioned on processed carries, all processed collision equations reduce to linear constraints on the frontier bits \(u\in U_k\). Store these in **canonical RREF**.

So the DP state is:
\[
S_k = (\gamma_k,\ \mathrm{RREF}(A_k\mid b_k)).
\]

Two different partial assignments that induce the same \(S_k\) are equivalent for all future extensions. That is the whole win.

## Transition

At bit position \(k\), the fresh decision variables are the 4 bits:
\[
x_k = (x_{57,k},x_{58,k},x_{59,k},x_{60,k}).
\]

Given current state \(S_k\), a local transition:

1. computes the corresponding \(W_2\) bits at slice \(k\),
2. evaluates all local bit-slice constraints for rounds 57..63,
3. branches only on the genuinely free local carry ambiguities,
4. updates the carry signature \(\gamma_{k+1}\),
5. appends the resulting linear equations to the residual system,
6. Gaussian-eliminates to new canonical state \(S_{k+1}\),
7. rejects contradictions.
