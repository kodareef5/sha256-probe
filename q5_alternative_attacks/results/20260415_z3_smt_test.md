# Z3 SMT Bit-Vector Collision Finder — Test at N=8

Z3 solver (z3-solver 4.16.0) with native bit-vector encoding.
Full cascade + collision encoded as BitVec(8) constraints.
32 free variables (W1[57..60] × 8 bits each).

Result: SAT in 284.1s. First collision: W57=0x7b W58=0x1f W59=0x0c W60=0x38.
Enumeration via blocking clauses: ~284s per solution → ~20h for all 260.

Comparison:
- Kissat (SAT): ~4s at N=8 (CSA encoding)
- Z3 (SMT): 284s at N=8 (bit-vector encoding)
- Ratio: Z3 is ~70x slower than Kissat

Conclusion: Z3's bit-vector theory doesn't help for collision finding.
The bitvector encoding creates the same search problem as SAT.
Z3 may be useful for sr=61 UNSAT proofs (theory-guided reasoning).
